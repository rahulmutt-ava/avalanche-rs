## 2. Goal / invariant to establish

Go's sender/router invariant, end-to-end in the live node: **every registered
request yields exactly one of {response, `*Failed`} delivered to the owning
engine** â€” within the adaptive timeout for silent loss, and *immediately* for
sends the network layer reports as not delivered.

## 3. Suspects (ranked; the RED test discriminates)

1. **Missing Go-parity "unsent â‡’ immediate `*Failed`" leg** in
   `OutboundSender::send_to`: Go's `sender.Send*` compares the set it
   registered with the set `network.Send` actually queued to, and immediately
   synthesizes `*Failed` for the difference. Our `send_to` is fire-and-forget
   (`sender.rs:131-140`) â€” go1's unsent request relies entirely on the timer
   backstop.
2. **Timer backstop dead on the live path**: RealClock `monotonic` â†”
   `tokio::time::sleep_until` interplay in `dispatch_loop`, or the fired
   handler's `handler_for(chain)`-captured-`None`/push routing of synthesized
   ops. Unit-tested only under MockClock.
3. (Known, related, separate) `ChainRouter::on_response` is dead code â€” timers
   are never cancelled on response. Benign for correctness today (phase guards
   drop late `*Failed`s) but couples with (2) and pollutes latency accounting.

Both (1) and (2) must end fixed regardless of which the test fingers first:
(1) alone unsticks this stall, but (2) is the only defense against silent
response loss after a *successful* send.

## 4. Approach

**Chosen: reproduce-then-fix at the production-wiring level (TDD).**
Build an in-process integration test through the real stack â€” shared
`ChainRouter` + `AdaptiveTimeoutManager` (RealClock, short configured
timeout) + `OutboundSender` over a mock `Network` + registered chain sink +
real `Bootstrapper` with N beacons â€” where one beacon's request is never
answered (and, variant, never even sent: mock `Network` drops it). Assert the
bootstrapper receives the `*Failed` and frontier discovery completes. The
failing link is then patched minimally, Go-parity documented
(`~/avalanchego` `snow/networking/sender/sender.go` as oracle).

Alternatives rejected:
- *Fix by inspection*: two plausible broken links; guessing risks a fix that
  passes review but not the live arm.
- *Sidestep by retrying the frontier broadcast on a timer*: masks the broken
  invariant; diverges from Go's architecture.
