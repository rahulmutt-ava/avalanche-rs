# M9.15 Live `mixed_network` Validation (firewood v0.7.0) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump the Rust firewood pin to v0.7.0 to match a rebuilt Go oracle at `d8a8473b`, then run the live two-binary `differential::mixed_network` arm to validate that the merged bootstrap-failure-accounting + beacon-gate-race fixes make the Rust follower bootstrap P/X/C from real Go.

**Architecture:** Three ordered parts. (A) A dependency bump verified against existing firewood goldens — TDD via the goldens that already exist. (B) An out-of-tree Go oracle rebuild + a light spec-pin fold. (C) A live run-and-observe loop that branches into a nested design→plan→fix sub-cycle if a fresh production blocker surfaces (the established "rung" pattern).

**Tech Stack:** Rust (cargo, cargo-nextest, cargo update), firewood-go-ethhash ffi (git dep), Go 1.25.10 (mise) for the avalanchego oracle, Task runner (`./scripts/run_task.sh`), Bazel (crate_universe / MODULE.bazel.lock).

## Global Constraints

- **License header** on every touched/created `.rs`: `// Copyright (C) 2019, Ava Labs, Inc. All rights reserved.` / `// See the file LICENSE for licensing terms.`
- **CI always calls a Task**, never `cargo`/`bazel` directly: `./scripts/run_task.sh <task>`. Single-crate iteration may use `cargo nextest run -p <crate>` directly.
- **BUILDABLE-&-GREEN invariant:** after Part A, full `./scripts/run_task.sh test-unit` + `lint-all` must pass before touching the oracle.
- **Firewood target version:** `v0.7.0` (git tag), replacing `v0.6.0`, in all four declarations.
- **Oracle toolchain:** rebuild `~/avalanchego` with **mise go 1.25.10** on PATH + GOROOT (nix go 1.26.3 silently fails to relink — memory `go-avalanchego-oracle-binary`). Gate on `./scripts/check_oracle_binary.sh` printing `OK` + asserting `rpcchainvm=45`.
- **Root-move policy:** an *unexpected* ethhash-root change is stop-and-investigate, not auto-regenerate. Regenerate a golden only when v0.7 legitimately changed hashing, and record provenance.
- **No commit signing** (`commit.gpgsign=false` is set locally). No raw `as` casts in `ava-saevm*` (not touched here).
