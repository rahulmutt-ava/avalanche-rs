# ava-simplex — porting notes (M3.17)

Port of avalanchego `simplex/` + `snow/consensus/simplex/parameters.go`
(specs 06 §8). Simplex is an alternative single-decree-per-round BFT consensus
(propose → vote → finalize with BLS-aggregated quorum certificates). This
milestone ports the **always-on wire surface** (parameters, canoto messages,
block wrapper, BLS quorum rules) and provides a **feature-gated engine stub**;
the full round-based BFT state machine is deferred.

## Canoto wire format (no codegen crate)

There is **no canoto codegen crate** in this workspace, so `src/canoto.rs`
hand-rolls the protobuf-flavoured canoto primitives (varint tags, `Len`
length-delimited byte fields, ascending field order, zero-value omission) and
the message types in `src/messages.rs` / `src/block.rs` encode/decode by hand to
round-trip **byte-identical** to Go's generated `qc.canoto.go` /
`block.canoto.go`. Field numbers/tags were read directly from the generated Go
files:

- `canotoQC`: `Sig` field 1 (fixed 96-byte BLS signature, tag `0x0a`),
  `Signers` field 2 (bitset bytes, tag `0x12`).
- `canotoSimplexBlock`: `Metadata` field 1 (`0x0a`), `InnerBlock` field 2
  (`0x12`), `Blacklist` field 3 (`0x1a`) — all `Len`-wire opaque bytes.

The signers bitset is Go's `set.Bits` = a `big.Int` serialized big-endian with
no leading zero bytes (empty set → empty slice). `signers_from_bytes` enforces
the canonical-form round-trip (rejects non-minimal encodings) exactly as Go's
`QCDeserializer.signersFromBytes` does via `bytes.Equal(indices.Bytes(), in)`.

## Golden vector provenance

`tests/vectors/{qc_canoto.hex,block_canoto.hex}` were generated by a scratch Go
program in `/tmp/simplexgen` (NOT committed to this repo) that imports the
upstream `github.com/StephenButtolph/canoto v0.18.0` library and replicates the
generated `MarshalCanotoInto` logic for `canotoQC` / `canotoSimplexBlock`. The
output bytes were then **re-decoded with the real `canoto.Reader`** to confirm
wire correctness (a Go test asserting field numbers, lengths, and values). The
vectors encode:

- **QC**: `Sig` = the bytes `1..=96`; `Signers` = bitset `{0,1}` → `[0x03]`.
- **Block**: `Metadata` = `ProtocolMetadata{Version:1, Epoch:2, Round:3, Seq:4,
  Prev:0xAB×32}` (57 bytes); `InnerBlock` = `0xDEADBEEF`; `Blacklist` =
  `0x0000` (a representative opaque 2-byte payload — the canoto wrapper treats
  the blacklist as opaque bytes; the inner blacklist parse is exercised only by
  the deferred engine).

`golden::simplex_qc_roundtrip` and `golden::simplex_block_roundtrip` assert that
the Rust encoder reproduces these exact bytes and that the Rust decoder recovers
the same logical fields. The full BLS crypto path (real aggregate → `bytes()` →
`from_bytes()` → `verify()`) is separately exercised in `tests/qc.rs` with real
`blst` keys (the QC golden vector's `Sig` is arbitrary bytes, not a valid BLS
point, so the crypto round-trip uses freshly generated signatures).

## Deviations / findings

1. **BLS aggregation reuses `ava_crypto::bls`** (`aggregate_public_keys`,
   `aggregate_signatures`, `verify`) — no third-party BFT/quorum crate, per
   spec 06 §8. `simplex.Quorum(n) = floor(2n/3) + 1` (one node = one vote).
2. **`encode_message_to_sign` is hand-rolled** rather than going through
   `ava-codec`. Go's `encodedSimplexSignedPayload` is a fixed 3-field
   linearcodec struct (`NetworkID u32`, `ChainID [32]byte`, `Message []byte`)
   with a 2-byte codec-version prefix; `CodecVersion = warp.CodecVersion(0) + 1
   = 1`. The layout `version(2) ++ network_id(4) ++ chain_id(32) ++
   message_len(4) ++ message` (all BE) is reproduced directly to avoid an
   `ava-codec` dependency for this single struct. Swap to a derived codec type
   when the fx/codec wiring lands.
3. **`p2p.Simplex` proto envelope deferred.** Go's `messages.go` wraps QCs,
   votes, and finalizations in `p2p.Simplex` protobuf messages routed through
   `SimplexHandler::simplex(node, msg)`. `ava-message`'s proto types for
   `p2p.Simplex` are not yet wired, so this crate ports the consensus-affecting
   payloads (`QC`, `BlockHeader`, `Vote`, `Finalization`) and the stub's
   `simplex` op takes `&[u8]` (matching the `ava-engine` `SimplexHandler` shape,
   which itself takes raw bytes — see `ava-engine/tests/PORTING.md`). Decode the
   envelope here once the proto type exists.
4. **Engine stub delegates to `NoOpHandler`.** `SimplexEngine`
   (`#[cfg(feature = "simplex")]`) implements every `ava-engine` op-group trait
   by forwarding to an embedded `NoOpHandler` (a Rust trait-family can't be
   selectively mixed in, mirroring Go's embedded `noOp*Handler`); the blanket
   `Handler`/`AllGetsServer` impls then apply for free. The `simplex` op
   log-and-drops; `start`/`health_check` are no-ops returning a
   `{"engine":"simplex-stub"}` health blob. Off by default.
5. **No Snowball `Parameters`.** Simplex finality is a BLS quorum, so there is
   no `k`/`alpha`/`beta` schedule — only `max_network_delay` /
   `max_rebroadcast_wait` timing + `initial_validators`. `DefaultParameters` has
   an **empty** validator set (matching Go), so `Parameters::default().verify()`
   intentionally fails until validators are supplied.
6. **Clippy `-D warnings`:** the lib uses `checked_*`/`saturating_*` and
   `.get(..)` everywhere (no panicking index/slice or unchecked arithmetic) so
   the `arithmetic_side_effects` / `indexing_slicing` warn-lints are clean even
   under `-D warnings`. Integration test files carry the repo-standard
   `#![allow(unused_crate_dependencies, clippy::unwrap_used, …)]` header (cf.
   `ava-vm` test crates). `tests/engine.rs` places the `allow` **before** its
   `#![cfg(feature = "simplex")]` gate so the suppression survives the
   feature-off empty-crate build.

## Deferred (not gaps)

- The full BFT engine state machine (`engine.go`): epoch/round, leader election,
  view-change timeouts, QC accumulation/notarization, replication.
- `storage.go` (finalized block + QC persistence) and `storage.canoto.go`.
- Inner-block parsing and the `blockTracker` accept/reject tree (`block.go`'s
  `verifyAndTrackBlock`/`indexBlock`) — needs a VM `block.Parser`.
- Blacklist semantics (the external `simplex.Blacklist` updates/suspected-nodes
  logic) — round-trips opaquely through the canoto block wrapper here.
