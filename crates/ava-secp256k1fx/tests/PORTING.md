## Error model

The fx sentinels (`ErrTimelocked`, `ErrTooManySigners`, `ErrTooFewSigners`,
`ErrInputCredentialSignersMismatch`, `ErrInputOutputIndexOutOfBounds`,
`ErrWrongSig`, `ErrOutputUnspendable`, `ErrOutputUnoptimized`,
`ErrAddrsNotSortedUnique`, the wrong-type set, …) already live on the shared
`ava_vm::error::Error` enum (added at M3.14, per the plan note "the fx wrong-type
set re-exported by `ava-secp256k1fx`"). `error.rs` re-exports `Error`/`Result`
from `ava-vm` so the fx framework and downstream VMs see one error type.

The syntactic-validation sentinels with **no `ava-vm` analog** — `ErrNilInput`
(N/A: a `&Input` is never nil in Rust), `ErrNoValueInput`, `ErrNoValueOutput`,
`ErrInputIndicesNotSortedUnique`, `ErrNilCredential` — are surfaced via
`Error::InvalidComponent(<go sentinel text>)`.

## `verify_credentials` (07 §4.3) — bit-for-bit

Reproduces `fx.go`'s `VerifyCredentials` exactly:

1. `owner.locktime > clock.unix()` ⇒ `Timelocked` (Go `Clock.Unix()` — whole
   seconds, clamped ≥ 0).
2. `threshold < num_sigs` ⇒ `TooManySigners`; `threshold > num_sigs` ⇒
   `TooFewSigners` (two branches, same order as Go).
3. `num_sigs != cred.sigs.len()` ⇒ `InputCredentialSignersMismatch`.
4. `!bootstrapped` ⇒ skip signature verification (`Ok(())`).
5. For each `(index, sig)`: `index >= len(addrs)` ⇒
   `InputOutputIndexOutOfBounds`; recover `pk` from
   `sha256(unsigned_tx.bytes())` (Go `hashing.ComputeHash256`) and require
   `addrs[index] == ripemd160(sha256(compressed_pk))` (`pk.address()`), else
   `WrongSig`. `cred.sigs[i]` pairs positionally with `sig_indices[i]`.

Signature recovery: `ava_crypto::secp256k1::PublicKey::recover_from_hash`
(low-S enforced before recovery, `00` §4.3). The `Fx` carries no recover cache
(the Go `RecoverCache` is a pure LRU optimization; recovery is deterministic).

## Golden vectors — `tests/vectors/secp256k1fx/vectors.json`

Generated by a scratch Go program built **against the pinned `../avalanchego`
tree** (run in a temp dir, not committed) and captured into `vectors.json`. The
program:

* `codec` array — `codec.Manager.Marshal(0, &concrete)` for each type (2-byte
  codec-version prefix `0x0000` + the `serialize:"true"` fields, no typeID —
  the same bytes the Go unit tests assert). The `transfer_input`,
  `transfer_output`, and `credential` vectors reproduce the exact `expected`
  byte arrays in `transfer_input_test.go` / `transfer_output_test.go` /
  `credential_test.go`.
* `multisig` array — real RFC6979 signatures over a fixed unsigned-tx byte
  string, with deterministic owner keys. **Each case's expected outcome was
  validated by driving the real `secp256k1fx.Fx.VerifyCredentials` in the
  generator before capture** (the program panics on any divergence), covering:
  threshold 1/2/3 accept, matured locktime, `ErrTimelocked`,
  `ErrTooManySigners`, `ErrTooFewSigners`, `ErrWrongSig`,
  `ErrInputOutputIndexOutOfBounds`, and the bootstrap signature-skip.

RFC6979 deterministic signing means the Rust `PrivateKey::sign_hash` reproduces
Go's signatures bit-for-bit, so the proptest (`prop::multisig_verify`) signs
locally rather than from captured sigs and still matches Go semantics.

## Tests

* `golden_codec.rs` — 8 byte-exact round-trip tests vs the committed vectors.
* `prop_multisig.rs` — `go_multisig_vectors` (10 Go-validated cases),
  `output_owners_verify_table` (`OutputUnspendable`/`OutputUnoptimized`/
  `AddrsNotSortedUnique`), and `prop::multisig_verify` (128 proptest cases:
  accept iff exactly `threshold` valid owner sigs at sorted-unique matured
  indices; reject over/under-sign, reordered, OOB, wrong-sig, premature).
